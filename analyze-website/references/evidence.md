# Evidence Rules

Use this file to keep website analyses disciplined.

## Source priority

Use sources in this order:

1. Official company pages
2. Official docs, pricing, API, help, blog, about, press
3. Reputable traffic and keyword tools, with Semrush preferred when directly accessible
4. Major third-party review sites
5. News coverage or market databases

If two sources disagree, prefer the more direct and more recent one.

## What must be verified

Verify with current sources when discussing:

- pricing
- traffic
- review counts
- ownership
- acquisition history
- monthly user claims
- employee count
- current integrations

Do not rely on memory for these.

When a Semrush page is accessible, prefer reading the live page over relying on search result snippets that summarize Semrush or Similarweb data.

## Revenue estimation guardrails

When estimating revenue:

- state clearly that it is an estimate
- show the inputs used
- use a range, not a single precise number
- separate self-serve revenue from enterprise revenue when possible
- anchor assumptions to packaging and observable scale claims

Avoid pretending certainty around:

- conversion rate
- ARPU
- enterprise contract value
- churn

## Writing discipline

- Distinguish fact from interpretation.
- Prefer exact dates over vague recency language.
- Use cautious language for uncertain claims: `likely`, `suggests`, `I infer`, `appears`.
- If a section lacks evidence, say `public information is limited` and keep moving.
- Preserve scope on third-party metrics: market, device, page type, and visible report date.

## Browser workflow

When browser tooling is available, collect these pages before writing:

- homepage
- pricing
- about
- integrations / API
- enterprise / case studies
- reviews if linked

This usually gives enough evidence to draft the report without over-browsing.

If the analysis needs traffic or SEO depth, add:

- Semrush domain overview
- Semrush backlinks overview
- Semrush AI visibility overview if available
- Semrush organic overview when SEO is central to the thesis
